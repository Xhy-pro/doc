# 通用 RAG 问答子技能

## 目的

本参考文档用于处理可以通过知识库回答的通用保险知识问题。

示例：

- 什么是等待期？
- 什么是免赔额？
- 重疾险和医疗险有什么区别？
- 理赔通常需要哪些材料？
- 退保是什么意思？

## 适用场景

当满足以下条件时，使用本子技能：

- 问题是通用问题，不绑定某个客户的个人保单或理赔案件。
- 答案可以由检索到的知识库资料支持。
- 不需要产品对比，也不需要复杂的产品条款推理。

## 不适用场景

不要在以下情况使用本子技能：

- 用户询问自己的保单、账户、缴费、理赔或案件。
- 用户要求个性化购买建议。
- 用户要求产品之间的对比。
- 用户询问违法、欺诈或规避规则的做法。
- 已完成最多三轮通用检索但证据仍不足以支持确定性回答。

## 输入要求

回答前先识别：

- 保险主题。
- 请求是通用问题还是客户个人问题。
- 原始用户问题。
- 当前已检索到的证据。
- 当前证据还缺少哪些信息才能回答原始问题。
- 证据是否充分、部分支持、缺失或冲突。

## 处理流程

1. 将问题分类为通用知识问题。
2. 保存用户的原始问题，后续所有检索都服务于回答这个原始问题。
3. 按 `rag_http_tool.md` 调用 HTTP RAG 工具进行第一轮检索，`intent` 使用 `general_qa`。
4. 验证第一轮证据是否足以回答原始问题。
5. 如果证据不足，先判断缺口是什么，再自主构建第二轮检索问题。
6. 第二轮后仍不足时，再基于前两轮证据缺口构建第三轮检索问题。
7. 最多只检索三轮。第三轮后仍不足时，停止检索并进入证据不足处理。
8. 综合所有轮次中的可用证据回答用户原始问题。
9. 如果证据只支持部分内容，只回答被支持的部分，并说明未确认内容。
10. 如果证据冲突，使用主技能的失败策略。
11. 当答案可能因产品、地区或官方规则不同而变化时，补充简短提醒。

## 多轮检索策略

通用检索不是简单重复同一个 query。每一轮都必须有明确目的：

1. 第一轮：直接检索用户原始问题。
2. 第二轮：针对第一轮无法回答的缺口，改写或拆分检索问题。
3. 第三轮：针对仍然缺失的关键条件、定义、例外或流程细节补充检索。

构建下一轮检索问题时，可以使用以下方法：

- 提取核心概念：例如等待期、免赔额、犹豫期、现金价值、健康告知。
- 拆分复合问题：例如“重疾险和医疗险区别以及怎么理赔”应拆成“区别”和“理赔材料/流程”。
- 补充同义表达：例如“报销比例”可补充检索“赔付比例”“给付比例”“补偿比例”。
- 聚焦缺失维度：例如第一轮只找到定义，第二轮检索适用条件、例外情况或常见限制。
- 保持原始意图：下一轮检索必须服务于原始问题，不要偏离到无关知识。

禁止：

- 三轮之后继续检索。
- 用同一个 query 机械重复检索。
- 用模型常识填补三轮后仍缺失的保险规则。
- 只根据最后一轮证据回答，忽略前面轮次的有效证据。

## 多轮检索记录

内部应维护一个简洁的检索记录，帮助综合回答：

```text
原始问题:
  用户最初提出的问题。

第 1 轮检索:
  query:
  已获得证据:
  仍缺少:

第 2 轮检索:
  query:
  已获得证据:
  仍缺少:

第 3 轮检索:
  query:
  已获得证据:
  仍缺少:

综合判断:
  哪些内容可回答，哪些内容不能确认。
```

不要把这份内部记录完整展示给用户。

## HTTP RAG 调用要点

- `query` 使用用户问题的核心保险概念，不要夹带无关对话。
- 第二轮和第三轮的 `query` 应体现上一轮的证据缺口，而不是重复原问题。
- `filters.insurance_types` 可放入明确的险种，例如重疾险、医疗险、寿险、车险。
- 不要把个人保单号、手机号、身份证号或案件号发送给 RAG 工具。
- 如果 RAG 工具失败或返回空证据，不要使用模型记忆补答。

## 回答要求

回答应当：

- 先直接回答问题。
- 综合多轮检索中所有有效证据，回答用户的原始问题。
- 用通俗语言解释保险术语。
- 避免不必要的法律化表达。
- 在相关时说明具体产品或实际业务处理可能不同。
- 不要假装已经完成任何业务操作。

## 澄清触发条件

出现以下情况时，只问一个澄清问题：

- 用户使用的术语有歧义。
- 用户说“这个保险”或“那个产品”，但没有提供名称。
- 用户询问的规则可能因产品类型不同而明显不同。
- 用户在一个问题中混合多个主题，直接回答会造成混乱。

## 安全回答示例

用户：“什么是等待期？”

安全回答模式：

```text
等待期通常是指保险合同生效后的一段期间，在这段期间内，某些保险责任可能尚未完全适用。根据当前知识库资料，如果保险事故发生在等待期内，具体处理结果通常要看产品条款约定。涉及具体产品时，应以该产品条款为准。
```

用户：“过了等待期保险就一定赔吗？”

安全回答模式：

```text
不一定。过了等待期只代表时间条件可能已经满足，是否能获得赔付还要看保险事故是否属于保障责任、是否触及免责条款、理赔材料是否完整，以及具体产品条款如何约定。
```

## 输出质量检查清单

- 回答有证据支持。
- 最多只进行了三轮检索。
- 第二轮/第三轮检索针对证据缺口构建，而不是机械重复。
- 回答综合了多轮有效证据，并回到用户原始问题。
- 回答不包含个人保单结论。
- 回答不包含个性化购买建议。
- 证据不足时明确说明不确定性。
- RAG 工具失败时没有伪造知识库结果。
- 非专业用户也能理解。
