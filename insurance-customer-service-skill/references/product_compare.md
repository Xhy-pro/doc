# 产品对比子技能

## 目的

本参考文档用于基于检索到的知识库证据，对两个或多个保险产品进行客观对比。

## 适用场景

当用户询问以下内容时，使用本子技能：

- 对比产品 A 和产品 B。
- 这些产品有什么区别？
- 哪个产品保障范围更广？
- 哪个产品等待期更短？
- 它们的免责条款有什么不同？
- 对比保障责任、给付规则、缴费期间或理赔条件。

## 不适用场景

不要使用本子技能来：

- 推荐用户购买哪个产品。
- 判断哪个产品更适合用户本人。
- 将产品做普遍性的最好/最差排名。
- 推断用户健康状况、预算、家庭需求或风险偏好。
- 承诺理赔结果。
- 在检索证据不足时进行产品对比。

## 输入要求

对比前先识别：

- 产品名称。
- 用户要求的对比维度。
- 每个产品对应的检索证据。
- 缺失的维度或缺失的产品资料。
- 用户是在要求客观对比，还是要求个性化推荐。

如果缺少产品名称或对比维度，只问一个澄清问题。

## 处理流程

1. 确认请求是客观对比。
2. 如果用户要求推荐，将问题重构为客观对比。
3. 按 `rag_http_tool.md` 调用 HTTP RAG 工具检索每个产品的证据，`intent` 使用 `product_compare`。
4. 只对证据支持的维度进行对比。
5. 表格能提升清晰度时，使用表格。
6. 对没有证据支持的维度，标注“当前知识库未找到”，不要猜测。
7. 结尾保持中立，说明实际购买决策应结合个人需求以及官方/具备资质的建议。

## HTTP RAG 调用要点

- 对每个产品分别检索，避免把 A 产品证据误用于 B 产品。
- `filters.product_names` 必须来自用户明确提供的产品名。
- `context.comparison_dimensions` 填入用户要求的维度，例如保障责任、等待期、免责、免赔额。
- 如果某个产品或某个维度没有证据，在对比表中标注“当前知识库未找到”。
- 如果 HTTP RAG 工具失败，不要继续生成看似完整的对比表。

## 对比维度

只使用证据支持的维度，例如：

- 产品类型。
- 保障责任。
- 免责条款。
- 等待期。
- 免赔额。
- 给付金额或赔付规则。
- 保险期间。
- 缴费期间。
- 投保年龄范围。
- 续保规则。
- 理赔材料或理赔条件。

## 推荐边界

如果用户问“我该买哪个”“哪个更适合我”或“哪个性价比更高”，不要替用户选择。

安全回答模式：

```text
我可以基于当前知识库资料做客观差异对比，但不能替你做个性化投保决策。下面先按保障责任、等待期、免责条款等维度对比；具体选择建议结合你的保障需求、预算、健康情况，并咨询官方或具备资质的顾问。
```

## 回答模式

推荐结构：

```text
根据当前检索到的资料，A 和 B 的主要差异如下：

| 对比维度 | 产品 A | 产品 B | 说明 |
| --- | --- | --- | --- |
| 保障责任 | ... | ... | ... |
| 等待期 | ... | ... | ... |
| 免责/限制 | ... | ... | ... |

当前知识库未能确认的部分：...

以上是基于公开资料/知识库资料的客观对比，不构成个性化投保建议。
```

## 澄清触发条件

出现以下情况时，只问一个问题：

- 用户只提供了一个产品名称。
- 用户说“A 和 B”，但上下文里没有明确产品名称。
- 用户问“哪个好”，但没有指定对比维度。
- 只有一个产品有证据，另一个产品缺少资料。

## 示例

用户：“A重疾险和B重疾险哪个更好？”

安全处理：

```text
我可以做客观差异对比，但不能直接判断哪个更适合你。你希望重点比较保障责任、等待期、免责条款、缴费方式，还是理赔条件？
```

用户：“对比 A 医疗险和 B 医疗险的免赔额和报销比例。”

安全处理：

```text
检索 A 和 B 的免赔额、赔付比例、适用条件和限制；只对证据覆盖的维度进行表格对比；缺失项标记为当前知识库未找到。
```

## 输出质量检查清单

- 对比基于证据。
- 未被证据支持的维度标记为不可用。
- HTTP RAG 工具失败时没有伪造对比内容。
- 没有给出个性化产品推荐。
- 没有声称某个产品普遍最好。
- 重要限制清晰可见。
